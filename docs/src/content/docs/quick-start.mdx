## Installation

Choose your preferred installation method:

<Tabs>
<TabItem label="Static Binary">

```bash
curl -sSL https://install.monosecret.dev | sh
```

</TabItem>
<TabItem label="Devenv.sh">

Add to your `devenv.nix`:

```nix
{ config, ... }:
{
  # Secrets are automatically populated from monosecret.toml
  env.DATABASE_URL = config.monosecret.secrets.DATABASE_URL;
  env.REDIS_URL = config.monosecret.secrets.REDIS_URL;
}
```

</TabItem>
<TabItem label="Nix">

```bash
nix-env -iA monosecret -f https://github.com/NixOS/nixpkgs/tarball/nixpkgs-unstable
```

</TabItem>

## Getting Started

Follow these steps to get started with Monosecret:

## 1. Initialize `monosecret.toml`

Discover secrets from existing `.env` files:

```bash
$ monosecret init
✓ Created monosecret.toml with 0 secrets

Next steps:
  1. monosecret config init    # Set up user configuration
  2. monosecret check          # Verify all secrets are set
  3. monosecret run -- your-command  # Run with secrets
```

If you have an existing `.env` file, you can import secrets from it:

```bash
$ monosecret init --from .env
```

Now edit `monosecret.toml` to define the secrets your application needs:

```toml
[project]
name = "my-app"
revision = "1.0"

[profiles.default]
DATABASE_URL = { description = "PostgreSQL connection string", required = true }
REDIS_URL = { description = "Redis connection string", required = false }

[profiles.development]
DATABASE_URL = { default = "sqlite://./dev.db" }

[profiles.production]
REDIS_URL = { required = true }
```

## 2. Set up provider backend

Configure your preferred secrets storage backend:

```bash
$ monosecret config init
? Select your preferred provider backend:
> keyring: Uses system keychain (Recommended)
  onepassword: OnePassword password manager
  dotenv: Traditional .env files
  env: Read-only environment variables
  pass: Unix password manager with GPG encryption
  protonpass: Proton Pass via official pass-cli
  lastpass: LastPass password manager
  gcsm: Google Cloud Secret Manager
  awssm: AWS Secrets Manager
  vault: HashiCorp Vault / OpenBao secret management
  bws: Bitwarden Secrets Manager
? Select your default profile:
> development
  default
  none
✓ Configuration saved to /home/user/.config/monosecret/config.toml
```

## 3. Import existing secrets (optional)

If you have existing secrets in environment variables or another `.env` file, you can import them:

```bash
# Import from current environment variables
$ monosecret import env

# Import from another .env file
$ monosecret import dotenv:/path/to/old/.env
```

## 4. Check and configure secrets

Verify that all required secrets are configured:

```bash
$ monosecret check
```

If any secrets are missing, you'll be prompted to set them. You can also set secrets manually:

```bash
$ monosecret set DATABASE_URL
Enter value for DATABASE_URL: postgresql://localhost/myapp
✓ Secret DATABASE_URL saved
```

## 5. Run your application

Run your application with secrets injected as environment variables:

```bash
$ monosecret run -- npm start

# Or with a specific profile and provider
$ monosecret run --profile production --provider dotenv -- npm start
```
